# Инструкция: плагин legal-brief для Claude Code

Плагин автоматически составляет процессуальные документы из папки с материалами дела. Вы указываете папку и говорите, что нужно написать — плагин сам читает все документы, составляет резюме дела, пишет черновик и прогоняет его через несколько раундов ревью и состязательной проверки.

---

## Шаг 1. Установить Claude Code

Если Claude Code ещё не установлен:

```bash
npm install -g @anthropic-ai/claude-code
```

Затем войти в аккаунт Anthropic:

```bash
claude
```

При первом запуске откроется браузер — авторизуйтесь через anthropic.com. После этого Claude Code готов к работе.

> Нужна подписка Anthropic с доступом к Claude API (план Pro или выше, либо API-ключ).

---

## Шаг 2. Установить плагин

Скопируйте папку `legal-brief` (ту самую, где лежит этот файл) в директорию плагинов Claude:

```bash
cp -r /путь/к/legal-brief ~/.claude/plugins/local/legal-brief
```

Либо клонируйте напрямую, если плагин размещён в git-репозитории:

```bash
git clone <url-репозитория> ~/.claude/plugins/local/legal-brief
```

Затем в Claude Code выполните:

```
/reload-plugins
```

Плагин появится в списке доступных команд.

---

## Шаг 3. Установить внешние зависимости

### 3a. Node.js и npm

Требуется Node.js версии 18 или выше.

```bash
node --version   # должно быть v18+
```

Если не установлен — скачать с [nodejs.org](https://nodejs.org) или через менеджер пакетов:

```bash
# macOS (Homebrew)
brew install node

# Ubuntu / Debian
sudo apt install nodejs npm
```

### 3b. Codex CLI

```bash
npm install -g @openai/codex
```

Проверить:

```bash
codex --version
```

Войти в аккаунт OpenAI (один раз, откроется браузер):

```
!codex login
```

> Префикс `!` перед командой в Claude Code означает «выполнить в вашем терминале». Введите это именно внутри сессии Claude Code.

После сообщения «Successfully logged in» Codex готов. Нужен аккаунт OpenAI с доступом к GPT-4 / Codex.

### 3c. Python и mark-all-the-shit-down

Требуется Python 3.10 или выше.

```bash
python3 --version   # должно быть 3.10+
```

Установить инструмент конвертации документов через официальный скрипт:

```bash
curl -sSf https://raw.githubusercontent.com/strigov/mark-all-the-shit-down/main/install.sh | sh
```

Скрипт сам скачает репозиторий, установит Python-зависимости (через `uv` или `pip`) и скопирует bundled tessdata для русского и английского языка. В конце должно появиться сообщение `mark-all-the-stuff-down installed`.

### 3d. Tesseract OCR (для сканов)

Нужен только если в деле есть сканы (JPG, TIFF, PNG). Для PDF и DOCX не требуется.

Устанавливать **языковые пакеты не нужно** — tessdata для `rus+eng` идёт в комплекте с mark-all-the-shit-down. Нужен только бинарь:

```bash
# macOS (Homebrew)
brew install tesseract

# Ubuntu / Debian
sudo apt install tesseract-ocr

# Windows (через Chocolatey)
choco install tesseract
```

Проверить:

```bash
tesseract --version
```

## Шаг 4. Запустить плагин

1. Откройте Claude Code в любой папке (рабочая директория не важна — папку дела вы укажете в диалоге).

2. Введите команду:

```
/legal-brief
```

3. Плагин задаст два вопроса:

   **Вопрос 1:** Укажите абсолютный путь к папке с материалами дела.  
   Пример: `/Users/ivan/Documents/дело_иванов_vs_петров`

   **Вопрос 2:** Что нужно составить?  
   Пример: `возражение на ходатайство о взыскании судебных расходов`

4. Дальше всё автоматически. Плагин пройдёт через 7 стадий и сообщит о каждой.

---

## Что происходит внутри

| Стадия | Что делает |
|--------|-----------|
| 0 — Конвертация | Все PDF, DOCX, сканы переводятся в текст (Markdown) |
| 1 — Резюме дела | Sonnet читает все материалы и составляет подробное резюме |
| 2 — Проверка | Codex проверяет точность резюме по исходным документам |
| 3 — Черновик | Codex пишет первый вариант документа |
| 4 — Ревью × 2 | Opus рецензирует, Codex правит — два прохода |
| 5 — Состязательная проверка | Opus-оппонент приводит контраргументы, Opus-судья выносит решение |
| 6 — Финальная доработка | Codex вносит правки с учётом решения судьи |

## Выходные файлы

Все результаты сохраняются в подпапку `_legal_brief/` внутри вашей папки с делом:

| Файл | Содержимое |
|------|-----------|
| `case_summary.md` | Резюме дела (стороны, требования, хронология, доказательства) |
| `draft_v1.md` | Первый черновик документа |
| `draft_v2.md` | Черновик после первого прохода ревью |
| `final_doc.md` | Документ после второго прохода ревью |
| `opponent_args.md` | Контраргументы оппонента |
| `judge_decision.md` | Решение судьи с рекомендациями |
| `final_doc_v2.md` | Итоговый документ |
| `state.json` | Состояние пайплайна (для возобновления) |

Сконвертированные `.md`-копии исходных документов создаются рядом с оригиналами — это промежуточные артефакты, не финальные выходы.

## Если работа прервалась

Если сессия оборвалась (закрыли терминал, пропал интернет, вышла ошибка) — просто снова запустите:

```
/legal-brief
```

Укажите ту же папку с делом. Плагин найдёт `_legal_brief/state.json` и спросит: **«Продолжить с прерванного места или начать заново?»**

Выберите «Продолжить» — работа возобновится с той стадии, где остановилась.

## Частые проблемы

**`/legal-brief` не найдена**  
→ Убедитесь, что плагин лежит в `~/.claude/plugins/local/legal-brief` и выполнили `/reload-plugins`.

**`codex: command not found`**  
→ Установите Codex CLI: `npm install -g @openai/codex`. Если npm не найден — сначала установите Node.js.

**`codex login` не работает / браузер не открывается**  
→ Убедитесь, что используете команду `!codex login` (с восклицательным знаком) внутри Claude Code. Или выполните `codex login` напрямую в обычном терминале.

**Сканы не распознаются / ошибка OCR**  
→ Проверьте tesseract: `tesseract --list-langs | grep rus`. Если русского нет — установите языковой пакет (шаг 3d).

**PDF не конвертируется**  
→ Проверьте: `pip show mark-all-the-shit-down`. Если пакет не найден — установите заново. Для PDF иногда нужен `poppler`: `brew install poppler` (macOS) или `sudo apt install poppler-utils` (Linux).

**Стадия завершилась, но файл не появился**  
→ Плагин остановится и сообщит об ошибке. Проверьте `_legal_brief/conversion_errors.txt` — там будет лог сбоев конвертации.

---

## Требования к папке с делом

- Любые форматы: `.pdf`, `.docx`, `.doc`, `.jpg`, `.tiff`, `.png` — всё поддерживается.
- Сканы распознаются автоматически через tesseract с русским и английским языком.
- Подпапки **не** обходятся рекурсивно — только файлы в корне указанной папки.
- Минимум один документ должен успешно сконвертироваться, иначе пайплайн не запустится.
