# legal-brief — инструкция по установке

Плагин для Claude Code: автоматически составляет процессуальные документы из папки с материалами дела. Одна команда `/legal-brief` — и плагин сам читает все PDF/DOCX/сканы, строит резюме дела, пишет черновик, прогоняет его через несколько раундов ревью и состязательной проверки.

---

## Быстрая установка (macOS / Linux)

```bash
git clone https://github.com/KunDeal/legal-brief ~/legal-brief
cd ~/legal-brief
bash install.sh
```

Скрипт зарегистрирует команду `/legal-brief` в Claude Code и покажет, каких зависимостей не хватает.

После этого — перезапустите Claude Code (или в VSCode: **Cmd + Shift + P → Developer: Reload Window**) и введите `/legal-brief`.

## Зависимости

Установите всё, что `install.sh` пометил красным крестом. Зелёные галочки — уже готово.

### Claude Code

```bash
npm install -g @anthropic-ai/claude-code
claude  # авторизация через браузер
```

Нужен аккаунт Anthropic (план Pro или API-ключ).

### Node.js 18+

```bash
# macOS
brew install node

# Ubuntu / Debian
sudo apt install nodejs npm
```

### Codex CLI

```bash
npm install -g @openai/codex
codex login  # авторизация через браузер
```

Нужен аккаунт OpenAI с доступом к GPT-4o / Codex.

### Python 3.10+

```bash
# macOS
brew install python

# Ubuntu / Debian
sudo apt install python3
```

### mark-all-the-shit-down (конвертация документов)

```bash
curl -sSf https://raw.githubusercontent.com/strigov/mark-all-the-shit-down/main/install.sh | sh
```

Устанавливает инструмент в `~/.local/share/mark-all-the-stuff-down/`. Работает без дополнительных настроек — tessdata для русского и английского языка идёт в комплекте.

### tesseract (только для OCR сканов)

Нужен, если в папке с делом есть сканы (JPG, TIFF, PNG). Для PDF и DOCX — не требуется.

```bash
# macOS
brew install tesseract

# Ubuntu / Debian
sudo apt install tesseract-ocr
```

## Использование

```
/legal-brief
```

Плагин задаст три вопроса:

1. **Путь к папке с материалами дела** — абсолютный путь, например `/Users/ivan/дело_123`
2. **Что составить** — произвольный текст, например `возражение на ходатайство о взыскании судебных расходов`
3. **Тип судопроизводства** — арбитраж (АПК), суд общей юрисдикции (ГПК) или КАС

Дальше всё автоматически. Результаты сохраняются в подпапку `_legal_brief/` внутри папки дела.

### Выходные файлы

| Файл | Что внутри |
|------|-----------|
| `case_summary.md` | Резюме дела |
| `draft_v1.md` | Первый черновик |
| `draft_v2.md` | После первого прохода ревью |
| `final_doc.md` | После второго прохода ревью |
| `opponent_args.md` | Контраргументы оппонента |
| `judge_decision.md` | Решение судьи |
| `final_doc_v2.md` | Итоговый документ |

### Если работа прервалась

Запустите `/legal-brief` снова с той же папкой дела. Плагин найдёт `_legal_brief/state.json` и спросит: **«Продолжить или начать заново?»**

## Частые проблемы

**`/legal-brief` не найдена**
→ Запустите `bash install.sh` повторно и перезапустите Claude Code.

**`codex: command not found`**
→ `npm install -g @openai/codex`, затем `codex login`.

**Скан не распознаётся**
→ Установите tesseract: `brew install tesseract` (macOS) или `sudo apt install tesseract-ocr` (Linux).

**PDF не конвертируется**
→ Переустановите mark-all-the-shit-down:
```bash
curl -sSf https://raw.githubusercontent.com/strigov/mark-all-the-shit-down/main/install.sh | sh
```

**Подпапки не обрабатываются**
→ Это ожидаемое поведение: плагин читает только файлы в корне указанной папки, подпапки игнорирует.

## Что происходит внутри

| Стадия | Модель | Что делает |
|--------|--------|-----------|
| 0 — Конвертация | — | PDF/DOCX/сканы → Markdown |
| 1 — Резюме | Sonnet | Читает все материалы, строит резюме дела; оркестратор проверяет, что каждый файл упомянут (греп-воротина, до 2 перезапусков) |
| 2 — Проверка резюме | Codex | Сверяет резюме с источниками, включая частичное/поверхностное покрытие |
| 3 — Черновик | Codex | Составляет первый вариант с учётом типа документа и процессуального типа |
| 4 — Ревью × 2 | Sonnet→Codex (проход 1), Opus→Codex (проход 2) | Рецензирует и правит, два прохода; применятор сверяется с резюме дела |
| 5 — Состязательная проверка | Opus | Оппонент + судья |
| 6 — Финальная доработка | Codex | Вносит правки по решению судьи, сверяясь с резюме дела |
| 7 — Предподачные проверки | Codex | Реестр ссылок (НПА + практика) + чек-лист реквизитов |
