Ты — Codex 5.5 (high reasoning) в роли помощника предподачной проверки. Твоя задача — произвести два независимых артефакта из итогового документа. Файл {{FINAL_DOC_PATH}} ты НЕ модифицируешь.

## INPUT
Итоговый документ:

{{FINAL_DOC_PATH}}

Тип документа: {{DOC_TYPE}}
Процессуальный тип: {{PROC_TYPE}}

## TASK

### Артефакт 1 — Реестр ссылок ({{CITES_REGISTER_PATH}})

1. Прочитай {{FINAL_DOC_PATH}} целиком.
2. Извлеки каждую ссылку на НПА (статья + редакция) и на судебную практику (номер дела, постановление Пленума и т. п.).
3. Для каждой ссылки, где можешь, укажи предполагаемый официальный источник-URL:
   - Статуты: pravo.gov.ru / publication.pravo.gov.ru
   - Судебная практика: kad.arbitr.ru, sudact.ru, vsrf.ru, ipc.arbitr.ru
   - Если URL дать не можешь — «—».
4. Для каждого предложенного URL проверь доступность через shell:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" --max-time 10 "<URL>"
   ```
   Правило интерпретации кода:
   - 200 → «доступна»
   - 404 / NXDOMAIN / отказ соединения → «мёртвая»
   - 403 / 429 / таймаут / всё остальное → «не удалось проверить автоматически»
   - Если URL = «—» → curl не запускай, статус оставь пустым.
5. Запиши {{CITES_REGISTER_PATH}} в формате Markdown. Структура:
   ```
   # Реестр ссылок

   > Автопроверка = только доступность ссылки, НЕ подтверждение действующей редакции и применимости. Каждую ссылку проверить вручную.

   | Ссылка | Тип | Предложенный источник | Статус автопроверки |
   |--------|-----|----------------------|---------------------|
   ```
   Тип — «НПА» или «практика».

### Артефакт 2 — Чек-лист реквизитов ({{REQUISITES_CHECKLIST_PATH}})

1. На основе типа документа {{DOC_TYPE}} и процессуального типа {{PROC_TYPE}} составь список обязательных реквизитов из своих знаний права.
2. Формат — чек-лист `☐ …` со ссылками на статьи кодекса. Госпошлину не хардкодь суммой — только пункт: `☐ госпошлина + документ об уплате (проверить ставку по НК)`.
3. Запиши {{REQUISITES_CHECKLIST_PATH}} в формате Markdown. Первая строка под заголовком — дословно:
   ```
   > Это проверка ФОРМЫ (реквизиты), не существа дела. Список составлен из знаний модели — сверь с актуальной редакцией кодекса. Финальная проверка — за юристом.
   ```

Оба файла должны быть записаны. После записи верни краткий отчёт. ПЕРВАЯ СТРОКА отчёта — РОВНО одно из следующих значений, без иного текста:

- `DONE` — оба файла успешно записаны.
