# legal-brief

Мульти-агентный плагин Claude Code для составления процессуальных документов из папки с материалами дела (российское право). Одна команда `/legal-brief` проводит материалы через 7 стадий: конвертация → резюме дела → проверка → черновик → 2 прохода рецензирования → состязательная проверка (оппонент + судья) → финальная доработка.

Главный поток Claude — оркестратор: задаёт два вопроса («Путь к папке с делом?», «Какой документ составить?»), диспетчеризует субагентов (Sonnet, Opus) и Codex-задачи, ведёт файловую машину состояний в `_legal_brief/` и показывает результаты на русском.

## Внешние зависимости (устанавливаются один раз)

Плагин зависит от двух внешних инструментов, которые НЕ вендорятся:

### 1. mark-all-the-shit-down (конвертация документов, стадия 0)

Python CLI для конвертации PDF/DOCX/изображений и т. п. в Markdown.

```bash
pip install mark-all-the-shit-down
```

Требует `tesseract` с языковыми пакетами «русский» и «английский» для OCR сканов:

```bash
# macOS (Homebrew)
brew install tesseract tesseract-lang
# проверка наличия русского пакета:
tesseract --list-langs | grep -E 'rus|eng'
```

OCR выполняется инструментом внутренне с флагами `-l rus+eng`. Точка входа резолвится как `python -m mark_all_the_shit_down.convert`, либо CLI `matsd`, либо `python convert.py` в каталоге установки.

### 2. codex CLI (бэкенд Codex-задач, стадии 2/3/4/6)

```bash
npm install -g @openai/codex
```

Логин один раз в своём шелле (браузерный OAuth):

```
!codex login
```

Префикс `!` в Claude Code делает это командой пользовательского шелла, а не вызовом инструмента. После «Successfully logged in» Codex готов.

Companion runtime для Codex вендорится внутри плагина (`vendor/codex-companion/`) и вызывается через обёртку `bin/codex-dispatch`; сам `codex` CLI — внешняя зависимость, через которую runtime работает. Состояние Codex изолировано в `~/.claude/plugins/data/legal-brief-codex`.

## Использование

```
/legal-brief
```

Ответь на два вопроса. Плагин создаст подпапку `_legal_brief/` в папке дела и запишет туда все артефакты (`case_summary.md`, `draft_v1.md`, …, `final_doc_v2.md`) и файл состояния `state.json`. Прерванную работу можно продолжить: при повторном запуске плагин предложит «Продолжить или начать заново».

## Выходные файлы (в `<папка дела>/_legal_brief/`)

| Файл | Описание |
|---|---|
| `state.json` | Состояние конвейера для возобновления |
| `case_summary.md` | Резюме дела |
| `draft_v1.md` | Первый черновик |
| `review_1.md` / `draft_v2.md` | Рецензия и черновик после прохода 1 |
| `review_final.md` / `final_doc.md` | Финальная рецензия и документ после прохода 2 |
| `opponent_args.md` | Контраргументы оппонента |
| `judge_decision.md` | Решение судьи |
| `final_doc_v2.md` | Итоговый документ |
| `conversion_errors.txt` | Лог ошибок конвертации (только при сбоях) |

Сконвертированные `.md`-файлы материалов создаются рядом с оригиналами (промежуточные артефакты, не финальные выходы).
