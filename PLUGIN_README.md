# legal-brief

Мульти-агентный плагин Claude Code для составления процессуальных документов из папки с материалами дела (российское право). Одна команда `/legal-brief` проводит материалы через 8 стадий (0–7): конвертация → резюме дела → проверка → черновик → 2 прохода рецензирования → состязательная проверка (оппонент + судья) → финальная доработка → предподачные проверки (реестр ссылок + чек-лист реквизитов).

Главный поток Claude — оркестратор: уточняет тип документа и процессуальный тип (арбитраж / ГПК / КАС), диспетчеризует субагентов (Sonnet, Opus) и Codex-задачи, ведёт файловую машину состояний в `_legal_brief/` и показывает результаты на русском.

## Внешние зависимости (устанавливаются один раз)

Плагин зависит от двух внешних инструментов, которые НЕ вендорятся:

### 1. mark-all-the-shit-down (конвертация документов, стадия 0)

Python-инструмент для конвертации PDF/DOCX/изображений и т. п. в Markdown. Устанавливается через официальный скрипт (клонирует репозиторий, устанавливает Python-зависимости, копирует bundled tessdata):

```bash
curl -sSf https://raw.githubusercontent.com/strigov/mark-all-the-shit-down/main/install.sh | sh
```

Скрипт устанавливает инструмент в `~/.local/share/mark-all-the-stuff-down/`. Плагин вызывает его напрямую: `python3 ~/.local/share/mark-all-the-stuff-down/convert.py`.

Для OCR сканов также требуется системный бинарь `tesseract` (языковые пакеты не нужны — tessdata для `rus+eng` идёт в комплекте):

```bash
# macOS (Homebrew)
brew install tesseract

# Ubuntu / Debian
sudo apt install tesseract-ocr
```

### 2. codex CLI (бэкенд Codex-задач, стадии 2/3/4/6)

```bash
npm install -g @openai/codex
```

Логин один раз в своём шелле (браузерный OAuth):

```
!codex login
```

Префикс `!` в Claude Code делает это командой пользовательского шелла, а не вызовом инструмента. После «Successfully logged in» Codex готов.

Companion runtime для Codex вендорится внутри плагина (`vendor/codex-companion/`) и вызывается через обёртку `bin/codex-dispatch`; сам `codex` CLI — внешняя зависимость, через которую runtime работает. Состояние Codex изолировано в `~/.claude/plugins/data/legal-brief-codex`.

## Использование

```
/legal-brief
```

Ответь на вопросы (путь к папке, тип документа, процессуальный тип). Плагин создаст подпапку `_legal_brief/` в папке дела и запишет туда все артефакты и файл состояния `state.json`. Прерванную работу можно продолжить: при повторном запуске плагин предложит «Продолжить или начать заново».

## Выходные файлы (в `<папка дела>/_legal_brief/`)

| Файл | Описание |
|---|---|
| `state.json` | Состояние конвейера для возобновления |
| `case_summary.md` | Резюме дела |
| `draft_v1.md` | Первый черновик |
| `review_1.md` / `draft_v2.md` | Рецензия и черновик после прохода 1 |
| `review_final.md` / `final_doc.md` | Финальная рецензия и документ после прохода 2 |
| `opponent_args.md` | Контраргументы оппонента |
| `judge_decision.md` | Решение судьи |
| `final_doc_v2.md` | Итоговый документ |
| `citations_register.md` | Реестр ссылок на НПА и практику с автопроверкой доступности |
| `requisites_checklist.md` | Чек-лист обязательных реквизитов для данного типа и процессуального типа |
| `conversion_errors.txt` | Лог ошибок конвертации (только при сбоях) |

Сконвертированные `.md`-файлы материалов создаются рядом с оригиналами (промежуточные артефакты, не финальные выходы).
